Overview

What is Spreadsheet Sync?

Spreadsheet Sync lets you connect a provider's existing Microsoft Excel file (stored in OneDrive or SharePoint) directly to the ATS platform. Once connected, you can run a sync at any time to:

⬇  Import candidates

New rows in the spreadsheet are automatically created as candidates in the ATS. Existing candidates are updated with the latest data from Excel.

⬆  Write back ATS status

After importing, the system writes current work status, pipeline stage, placement info, and welfare check results into four dedicated ATS columns in your spreadsheet.

How it works — at a glance

📊
OneDrive / SharePoint Your Excel file with jobseeker rows
⟷
🔄
Spreadsheet Sync Reads rows, upserts candidates, writes back ATS columns
⟷
🗄️
ATS Database Candidates, applications, placements, welfare checks
🔐
Access control Only users with the Admin or Recruiter Admin role can connect OneDrive, configure the spreadsheet, and trigger syncs. Regular recruiter accounts can view sync logs but cannot initiate a sync.
⚠️
One spreadsheet per provider Each provider can have one connected spreadsheet. If a candidate's email already exists in the ATS under a different provider, that row will be skipped during sync.

Before you start

Prerequisites & Access

Before running your first sync, make sure the following are in place:

  • Admin or Recruiter Admin role — Your ATS account must have one of these roles. Check with your system administrator if unsure.
  • A provider record exists — The employment service provider must already be created in the ATS (Providers list page). If not, an Admin can add it first.
  • Your spreadsheet is in OneDrive or SharePoint — The Excel file must be stored in a Microsoft 365 OneDrive or SharePoint account that you have access to sign in with.
  • Required columns are present in the spreadsheet — At minimum, your spreadsheet must have a header row containing email and jobseeker columns (exact names, case-insensitive). See the full column reference on page 9.
  • Sharing link for the file — You will need a copy or share link to the Excel file from OneDrive. You can get this by right-clicking the file in OneDrive and selecting Share → Copy link.
💡
Which Microsoft account to use? Sign in with the Microsoft 365 account that owns (or has Editor access to) the spreadsheet. This is usually a work/school Microsoft account, not a personal outlook.com account.

Step 2

Connect Your OneDrive Account

The first time you set up sync for a provider, you need to authorise the ATS to access OneDrive on behalf of the provider's Microsoft account.

  1. In the Spreadsheet Sync panel, click the Connect OneDrive button.
  2. You will be redirected to the Microsoft login page in the same browser tab.
  3. Sign in with the Microsoft 365 account that has access to the provider's Excel file.
  4. On the permissions screen, review the requested permissions and click Accept.
  5. You will be automatically redirected back to the Provider Detail page.

Step 3

Link the Spreadsheet File

After OneDrive is connected, you need to tell the ATS which Excel file to sync. You do this by pasting a share link from OneDrive.

Getting the OneDrive link

  1. Open OneDrive in a new browser tab and locate the Excel file.
  2. Right-click (or click the three-dot menu) on the file.
  3. Choose Share → then click Copy link.
  4. Make sure the link is set to Anyone with the link or People in your organisation — not restricted to specific people only.
💡
SharePoint links work too If the file is stored in SharePoint, you can paste the SharePoint file URL directly. The system uses the same Microsoft Graph API to resolve both OneDrive and SharePoint links.

Saving the spreadsheet

  1. In the ATS, paste the link into the OneDrive spreadsheet URL field.
  2. If your data is not on the first sheet, update the Sheet name field (defaults to Sheet1).
  3. Click Save & Connect.

❌
Error: "Could not resolve OneDrive URL" This means the link is invalid or the signed-in Microsoft account doesn't have access to that file. Double-check the link and make sure the file sharing permissions allow access.
⚠️
Sheet name is case-sensitive The sheet name must exactly match the tab name in your Excel file. Sheet1 and sheet1 are different. Check the tab at the bottom of your Excel file.

Step 4

Run Your First Sync

With OneDrive connected and the spreadsheet file saved, you're ready to sync. The sync process reads every data row in your spreadsheet and processes it in the ATS.

  1. In the Spreadsheet Sync panel, click the Sync Now button.
  2. The button shows a spinning indicator while the sync is in progress — this usually takes 5–30 seconds depending on the number of rows.
  3. When complete, a green result banner appears with a summary of what changed.

When should I run sync? Sync is manual — run it whenever your spreadsheet has been updated with new or changed jobseeker information. There is no automatic schedule; you control when syncs happen.
⚠️
Don't navigate away during sync The sync runs in the browser tab. If you close the tab or navigate away before it finishes, the sync will still complete on the server, but you won't see the result message. Check the sync history log for the outcome.

Step 5

Review Sync Results

After each sync, the panel displays a history of the last five sync runs. Use this to verify what happened and catch any issues.

Understanding the result numbers

FieldWhat it means
Created Rows that did not exist in the ATS — new candidate records were created.
Updated Rows already in the ATS — the candidate's name, phone, notes, and other fields were refreshed from the spreadsheet.
Written back Rows where the ATS successfully wrote current status (work status, pipeline stage, placement, welfare check) back into the spreadsheet's ATS columns.
Skipped Rows that were not processed — either no email AND no name/phone to generate a placeholder, or the email belongs to a candidate under a different provider.

Sync history log

localhost:5173/providers/abc-123
Spreadsheet Sync

Recent Syncs

✓ 31 May 9:14 am — 5 created, 3 updated, 8 written back
✓ 30 May 2:05 pm — 0 created, 2 updated, 2 written back
✗ 29 May 10:32 am — Failed: Missing required column: "email"
✓ 28 May 4:17 pm — 12 created, 0 updated, 12 written back, 2 skipped
⚠️
Rows showing as Skipped? A row is skipped if it has no email, no name, and no phone number (nothing to identify the person), OR if the email is already linked to a candidate under a different provider. Check those rows in your spreadsheet and add missing information.

What happens to candidates without an email address?

If a row has no email but does have a name or phone number, the system auto-generates a placeholder email in the format:

firstname-lastname-0412345678@sync.local

This ensures the candidate is still created in the ATS. You can update the email address on the Candidate Detail page later once you have it.

✅

These will be blank for brand-new candidates (no applications or placements yet) and will populate as activity is recorded in the ATS.

What if the candidate is missing or has wrong data?

Candidate doesn't appear in search
Check the sync log — if the row was skipped (shown in the Skipped count), it means the row had no email, name, or phone. Go back to the spreadsheet, add the missing data, and run the sync again.
Candidate exists but is linked to a different provider
Open the candidate and check the Provider field. If it shows a different provider, the email address was already registered under that provider before this sync. The row will have been skipped — you'll need to reconcile the duplicate in the ATS manually.
Fields are blank or have old values
The sync updates candidates based on the current spreadsheet content. If the field is blank in Excel, it will be synced as blank. Check your spreadsheet row and ensure the data is filled in, then run the sync again.
Email shows as name-phone@sync.local
The spreadsheet row had no email address, so a placeholder was generated. You can edit the candidate in the ATS and update the email to the real address once you have it. The next sync will recognise the real email and update the record.
💡
Spot-check tip After a large sync (50+ rows), pick 3–5 candidates at random from different parts of the spreadsheet — one near the top, one in the middle, one near the bottom — and verify each one. This catches any column-mapping issues that affect a whole section of the file.

What Gets Written Back to Excel

After importing candidates, the sync automatically adds (or updates) four ATS columns in your spreadsheet. These columns are created by the system if they don't already exist.

Column name added to spreadsheetWhat it contains
Work Status (ATS) The candidate's current work status in the ATS, e.g. job_seeking, employed, placed, inactive.
Pipeline Stage (ATS) Their current stage in the most recent job application pipeline, e.g. applied, screening, interview, offer, hired.
Placement (ATS) Employer name and placement start date for their most recent placement, e.g. Aldi Supermarkets – 2026-04-14.
Welfare Check (ATS) The type of the most recent completed welfare check, e.g. 4-week, 12-week, 26-week.

Columns are created automatically If the four ATS columns don't exist in your spreadsheet yet, the system creates them at the end of the existing columns the first time you run a sync. You don't need to add them manually.
⚠️
Do not rename or delete the ATS columns If you rename the four ATS columns, the next sync will treat them as missing and create new columns alongside them. The old (renamed) columns will be left as-is.

Required Spreadsheet Columns

Your spreadsheet's first row must be a header row. The sync engine reads column names case-insensitively and maps them to candidate fields. Below is the full column reference.

Required columns

Column nameMaps toNotes
email REQUIRED Candidate email Used as the unique identifier. If missing, a placeholder is generated from name + phone.
jobseeker REQUIRED Candidate full name The person's full name. Also used for the placeholder email if email is blank.

Recognised optional columns

Column nameMaps toNotes
mobile OPTIONAL Phone number Stored as-is. Also used in placeholder email generation.
ideal roles OPTIONAL Interested job type Free-text field for preferred role or industry.
car or pt? OPTIONAL Transport type Accepts: car, pt / public transport, or any combination for both.
wage sub - max $ & hours OPTIONAL Wage subsidy details Parses dollar amount (e.g. $10,000) and hours (e.g. 520 hrs).
comments - experience, hours etc OPTIONAL Candidate notes Free-text. Appended to the candidate's notes field.
ec OPTIONAL EC reference (notes) Appended to notes as [EC: value].
please refer to (ec only) OPTIONAL EC refer note Appended to notes as [EC Refer: value].
comments (ec only) OPTIONAL EC comments Appended to notes as [EC Comments: value].
have referred to (wv only) OPTIONAL WV refer note Appended to notes as [WV Refer: value].
comments (wv only) OPTIONAL WV comments Appended to notes as [WV Comments: value].
💡
Extra columns are ignored Any columns in your spreadsheet that aren't in the list above are simply ignored by the sync engine. You can have as many extra columns as you like without affecting the sync.

Maintenance

Disconnecting & Reconnecting

How to disconnect OneDrive

If you need to remove the OneDrive connection for a provider (e.g. the account changed, or you need to connect a different file), click the Disconnect button in the Spreadsheet Sync panel.

localhost:5173/providers/abc-123
Spreadsheet Sync
Connected as jane@provider.org.au
⚠️
What disconnecting does Disconnecting removes the stored Microsoft OAuth tokens and the spreadsheet file link from the provider record. It does not delete any candidate data that was previously imported. You can reconnect at any time.

Reconnecting after a disconnect

After disconnecting, the panel returns to the Not connected state. Follow Steps 2–3 again to reconnect: click Connect OneDrive, sign in to Microsoft, then paste the spreadsheet URL.

Token expiry & automatic refresh

Microsoft OAuth access tokens expire after one hour. The ATS automatically refreshes the token before each sync run using the stored refresh token. You do not need to reconnect periodically.

❌
Sync failed with a 401 Unauthorised error If the refresh token itself has expired (which can happen if the token is unused for 90+ days, or if the Microsoft admin revoked access), you will see this error. Simply disconnect and reconnect to generate fresh tokens.

Changing the spreadsheet or sheet name

If you move your data to a new Excel file or a different sheet tab:

  1. Click Disconnect in the Sync panel.
  2. Click Connect OneDrive and sign in again.
  3. Paste the new spreadsheet URL and update the sheet name.
  4. Click Save & Connect, then run a sync.

Troubleshooting

❌ "Provider spreadsheet not connected" error when I click Sync Now
OneDrive has been connected but no spreadsheet file has been saved yet. Go back to Step 3 and paste the OneDrive share link to configure the file.
❌ "Missing required column: email" or "Missing required column: jobseeker"
Your spreadsheet's header row (row 1) doesn't contain a column named email or jobseeker. Check the column names — they must match exactly (case doesn't matter, but spelling does). Rename the columns in Excel and run the sync again.
❌ "Spreadsheet is empty or has only a header row"
The Excel sheet has a header row but no data rows below it. Add at least one data row to the spreadsheet and sync again.
❌ "Could not resolve OneDrive URL — check the link is valid"
The share link you pasted doesn't resolve to an accessible file. Check: (1) the link is correct and not truncated; (2) the sharing is set to "Anyone with the link" or "People in your organisation"; (3) you're connected with the right Microsoft account.
⚠️ Many rows are showing as Skipped
Rows are skipped for two reasons: (a) the row has no email, name, or phone — add the missing data; or (b) the email belongs to a candidate under a different provider — this is expected and cannot be overridden.
⚠️ Candidates were created but their email looks like "john-smith-0412345678@sync.local"
These are placeholder emails auto-generated because the spreadsheet row had no email address. Update the candidate's real email address on their Candidate Detail page in the ATS once you have it.
⚠️ ATS columns were not written back to my spreadsheet
Check the sync log — if the sync shows rows written back as 0, it may be because the candidates have no applications, placements, or welfare checks yet. There's nothing to write back until activity is recorded in the ATS.
❌ Sync failed with "Graph GET error 401"
The Microsoft access token has expired and could not be refreshed. Disconnect and reconnect OneDrive from Step 2. This is typically caused by a long period of inactivity (90+ days without a sync) or an admin revocation of OAuth permissions.
🔒 The "Connect OneDrive" and "Sync Now" buttons aren't visible
These buttons are only visible to Admin and Recruiter Admin users. If you don't see them, your account role doesn't have sufficient permissions. Contact your ATS administrator to update your role.
